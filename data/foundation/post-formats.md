# Post Formats

## Rules That Always Apply

- Short sentences. Never 3 lines without a break.
- No filler words: "genuinely", "game-changer", "dive into", "unleash"
- Hook must work without "see more" — first 2 lines decide everything
- One idea per post — don't try to say everything
- End with a question or a statement, never both
- Human voice — sounds like Salim wrote it, not ChatGPT

## Anti-AI Writing Rules

- No symmetrical sentence pairs: "We stopped X. Started Y." — always sounds like ChatGPT
- No dot-fragments on the same line: "Personalization tokens. 4-step sequences. The breakup email." — choppy and artificial. Use a real sentence or a proper ladder instead
- No polished pivots or clean transitions — real writing is messier than that
- No em dashes (—) ever
- Format comes from the swipe file, not generic copywriting instincts — study the creator's actual posts before writing
- Hooks come from the chosen creator's style only — never invent from scratch

## Charles Tenot Style Rules

- Open with "We" + specific number or action (his most common and best-performing pattern)
- Medium-length hook (11–20 words) — his best avg likes
- Text-only — 1.6× more engagement than posts with images
- Conversational tone — "it feels awesome", "this tells a lot about", not polished copy
- Past tense storytelling: "We did X. Here's what happened."
- Format: Proof on yourself — milestone → mistake → what changed → lesson

## Ladder Structure (use for signal lists, step lists, recap blocks)

Use `→` bullets when listing items that belong together as a group — signals, steps, or a closing recap:

```
→ Item one
→ Item two
→ Item three
```

Never use this on the same line as running text (no dot-fragments). Each item gets its own line.

**Mid-post ladder** — use for a list of triggers, signals, or inputs:
```
If none of these signals were there, we didn't send:

→ New hire in a key role
→ Fresh funding
→ Tech stack change
→ Competitor switch
```

**Closing recap ladder** — use to summarize the system before the CTA:
```
Here's what we run now:
→ Signal detection before any email goes out
→ One angle per company, written for that company
→ 2-touch max, then we move on
```

## Format 1 — Hook Post
**Best for:** Contrarian take, surprising stat, bold claim

```
[Bold one-line hook]

[2-3 lines expanding on the hook]

[1-2 lines of proof or example]

[Closing statement or question]
```

---

## Format 2 — Story Post
**Best for:** Client results, personal experience, lessons learned

```
[Situation — where you/client started]

[Problem — what went wrong or what the challenge was]

[Turn — what changed]

[Result — specific numbers]

[Lesson — one takeaway]
```

---

## Format 3 — List Post
**Best for:** Practical tips, tool breakdowns, frameworks

```
[Hook — "X things I wish I knew about Y"]

1. [Point]
2. [Point]
3. [Point]
...

[Closing takeaway]
```

---

## Format 4 — Hot Take
**Best for:** High engagement, debate, visibility

```
[Controversial statement]

[Why most people think the opposite]

[Why you disagree — with specific reasoning]

[What you do instead]
```

---

## Format 5 — How-To
**Best for:** Education, authority building, saves

```
[Problem you're solving]

Here's exactly how I do it:

Step 1: [action]
Step 2: [action]
Step 3: [action]

[Result you get]

[CTA — save this / try it / what would you add?]
```
