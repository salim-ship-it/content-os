# Idea Capture

The central log for all content ideas. Everything flows through here before becoming a post.

## How Ideas Get Here

| Source | Workflow | Frequency |
|---|---|---|
| LinkedIn feed scanning | Workflow 2 (Daily Capture) | Daily |
| Creator post pulls via Apify | Workflow 1 (Pull & Categorize) | Weekly |
| Newsletter reading | Workflow 3 (Weekly Scan) | Monday |
| Reddit scanning | Workflow 3 (Weekly Scan) | Monday |
| Daily debrief (your own day) | `daily-debrief.md` | Daily |
| Active research (when stuck) | Workflow 5 (Active Research) | As needed |

## Idea Format

```
---
Date: YYYY-MM-DD
Source: [creator name / newsletter / subreddit / own experience]
Original: [link or description of what inspired it]
My angle: [how I'd adapt it for my audience]
Format: [hot take / story / case study / educational / insightful]
Status: idea / drafting / published / archived
Priority: 🔴 Post within 48h / 🟡 This week / 🟢 Evergreen
```

## Status Flow

```
idea → drafting → published → (log results)
  ↓
archived (if stale after 60 days)
```

## Rules

- **Minimum 1 idea per day** — aim for 2-3
- **Don't overthink the angle** — raw is fine, refine when you draft
- **Check for duplicates** before logging — scan recent entries first
- **Most recent ideas at the top** of the log
- **Mark status changes immediately** — don't let ideas sit as "idea" forever
- **🔴 ideas expire** — if not posted within 7 days, they lose their edge
- **After publishing:** log likes, comments, reposts + what worked (feeds back into `memory.md`)

## When to Promote to Winning Posts

If a post you publish gets **200+ likes or 20+ comments**, add it to the matching file under `post-database/winning-posts/` and update `memory.md` with what you learned.

## Connected Files

| File | Relationship |
|---|---|
| `workflows.md` | Step-by-step processes that feed ideas into this log |
| `sources.md` | Where to look for ideas |

| `post-database/winning-posts/memory.md` | What's working — check before picking which idea to write |
